# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please **do not open a public issue**. Report it directly via email:

**[juliocmbuenotrabalho@outlook.com](mailto:juliocmbuenotrabalho@outlook.com)**

Include:

* Description of the vulnerability
* Steps to reproduce (if possible)
* Environment details (Java version, module, etc.)

---

## Supported Versions

| Version | Status    |
| ------- | --------- |
| 1.x     | Supported |

---

## Security Practices

* Validate and sanitize user input when using dynamic projections.
* Limit exposure of sensitive fields in projections.
* Keep dependencies up to date.
* Use HTTPS for external services and repositories.
